Scrappie

I made this webapp because I want a more visually appealing way to easily bookmark recipes online. The name of this app is a play of the words between scrape book and web scrapping. To get the images from the recipe url, I used a web scraping library to get the HTML of the page. Then I use getElementsByTagName to get all of the img tag and after that filter out ad banners and other unwanted images. For account authentication, I utilised an library called PassportJS to enable the creation of accounts via email, facebook, twitter, or google. Data for the recipes that the user has saved are stored in a database as JSON format in a mlab (a cloud mongoDB database) and storage for the pictures are stored in Amazon S3.

To do app

I made this to do app because I couldn't find a good weekly to do app to use. I made use of the HTML drag and drop API so that the user can drag and drop the tasks between dates and folders. I also made a calender widget to enable the user to jump between dates. I used Firebase as my backend-as-a-service to handle the database storage and authentication. The to do tasks are stored in a single JSON format sepearated by the dates. Then authentication can handle email, Facebook, and Google accounts.

Tic Tac Toe

I made this app is to see if my algorithm skills are good enough to program a tic tac toe game that will always win against a human. There are 4 main moves the computer can take. An offensive move, where the computer can determine which grid to place in order to win the game. A defensive move, where the computer can determine which grid to place in order to block to user from winning the game. The last 2 moves are predetermined moves where it will always guarentee a win based on the starting position of the user.
