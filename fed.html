Outline Volunteer Roster

When I was volunteering for OUTline NZ, I realised that the process for registering my shift is inefficient. Knowing that this issue can be solved by a web app, I approached the interim manager of my skills and we came up with an online roster web app. I designed and built the web app from scratch using Sketch as my design tool and Polymer, a web component library. I've also used Firebase to handle the backend which includes authenication, database, and storage. We recently deployed the beta version of the web app for other volunteers to try out and use. Even though it has only been released recetnly, we have been getting positive feedback from not only the volunteers, but also from the management.

Clothes for kids

I participated in a weekend charity hackathon organised by ImpactNPO. My team and I were tasked to provide any digital solution to our chosen charity. The charity that my team chose was Clothes for Kids. Clothes for Kids is a volunteer based organisation helping to ease the financial burden on parents struggling to clothe their children. Since our team is quite big, consisting of 7 people. We decided that we could afford to split up to deliver two different solutions, a streamlined Google Form to help up new signups for volunteers and a website to boost the digital presence for the charity. My role is to design and build the static version of the website while the back end developer connects it to a CMS to allow the charity to update the copy in the future. With only less than 2 days to complete, we miraculously delivered a product that exceeds the charity's expectation.

To do app

I have been looking for a to do app to help me organise my weekly tasks but I realise that I could just make one for myself. I have also wanted to try out the HTML drag and drop API for a project so this is the perfect opportunity to do so. By making use of the HTML drag and drop API, the user can drag and drop the tasks between dates and folders. After hearing feedback from some of my friends, I also made a calendar widget to enable the user to jump between dates. I used Firebase as my backend-as-a-service to handle the database storage and authentication. The to do tasks are stored in a single JSON format separated by the dates. Then authentication can handle email, Facebook, and Google accounts.

Tic Tac Toe

This tic tac toe game is programmed to never lose. There are 4 main moves the computer can take. An offensive move, where the computer can determine which grid to place in order to win the game. A defensive move, where the computer can determine which grid to place in order to block to user from winning the game. The last 2 moves are predetermined moves where it will always guarentee a win based on the starting position of the user.
